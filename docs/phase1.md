# Groot --- Phase 1

## Goal

Implement minimal tenant management and event ingestion.

Events must be accepted via HTTP, validated against a tenant, and
published to Kafka.

No routing, subscriptions, or delivery logic.

------------------------------------------------------------------------

# Scope

Phase 1 implements:

1.  Tenants (PostgreSQL)
2.  Tenant API keys
3.  Tenant authentication middleware
4.  Event ingestion endpoint
5.  Canonical event envelope
6.  Kafka event publishing
7.  Minimal logging

------------------------------------------------------------------------

# Database

Create migration:

migrations/001_create_tenants.sql

Schema:

CREATE TABLE tenants ( id UUID PRIMARY KEY, name TEXT NOT NULL,
api_key_hash TEXT NOT NULL, created_at TIMESTAMP NOT NULL DEFAULT NOW()
);

Constraints: - id generated by application - api_key_hash stores hashed
key only

------------------------------------------------------------------------

# Tenant API Key

Tenant API keys must: - be generated on tenant creation - be returned
once - never stored in plaintext

Generation: - use cryptographically secure random bytes - encode using
base64 or hex

Storage: - hash using SHA256 before storing

------------------------------------------------------------------------

# Tenant API

Base path: /tenants

## Create Tenant

POST /tenants

Request:

{ "name": "example" }

Response:

{ "tenant_id": "uuid", "api_key": "generated_key" }

Notes: - API key returned only once - API key hash stored in database

------------------------------------------------------------------------

## List Tenants

GET /tenants

Response:

\[ { "id": "uuid", "name": "example", "created_at": "timestamp" }\]

------------------------------------------------------------------------

## Get Tenant

GET /tenants/{tenant_id}

Response:

{ "id": "uuid", "name": "example", "created_at": "timestamp" }

------------------------------------------------------------------------

# Authentication

All event ingestion requires tenant authentication.

Header: Authorization: Bearer `<api_key>`{=html}

Middleware responsibilities: 1. extract API key 2. hash key 3. lookup
tenant by api_key_hash 4. reject request if not found 5. attach
tenant_id to request context

Unauthorized requests return HTTP 401.

------------------------------------------------------------------------

# Canonical Event

Define struct:

internal/stream/event.go

Fields: event_id tenant_id type source timestamp payload

Types: - event_id → UUID - tenant_id → UUID - type → string - source →
string - timestamp → RFC3339 time - payload → raw JSON

Rules: - event_id generated server-side - timestamp generated
server-side - tenant_id comes from auth middleware - payload stored
exactly as received

------------------------------------------------------------------------

# Event Ingestion API

Endpoint: POST /events

Authentication: Authorization: Bearer `<api_key>`{=html}

Request body:

{ "type": "example.event", "source": "manual", "payload": {} }

Validation: - type required - source required

Server behavior: 1. read request 2. validate fields 3. build canonical
event 4. publish event to Kafka 5. return success if publish succeeds

Response:

{ "event_id": "uuid" }

HTTP status: - 200 success - 400 validation error - 401 auth failure -
500 publish failure

------------------------------------------------------------------------

# Kafka

Topic: events

Create topic if it does not exist.

All events are published to this topic.

------------------------------------------------------------------------

# Kafka Publisher

Implementation location: internal/stream

Responsibilities: - create producer - serialize event to JSON - publish
to topic events

Function signature example:

PublishEvent(ctx context.Context, event Event) error

Errors must propagate to HTTP handler.

------------------------------------------------------------------------

# Logging

Log events: tenant_created event_received event_published
event_publish_failed

Logs must include: - event_id - tenant_id - event_type

Do not log payload.
